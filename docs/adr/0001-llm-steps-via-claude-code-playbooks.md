# LLM-шаги пайплайна выполняются через плейбуки Claude Code, а не Anthropic SDK (MVP)

Спека закладывала `scripts/extract.ts` с Anthropic SDK как ядро пайплайна. До валидации идеи мы не готовы тратить деньги на API, поэтому в MVP LLM-шаги (извлечение, слияние) описаны как плейбуки — промпт-шаблоны в `prompts/` — и выполняются в сессиях Claude Code, покрытых подпиской. Детерминированные шаги (normalize, verify-quotes, validate) остаются обычными скриптами.

Контракт «выход LLM-шага — JSON-черновики в `content/drafts/`, фронтенд читает только провалидированный `content/`» не зависит от исполнителя, поэтому миграция на SDK — это замена исполнителя плейбуков при неизменных промптах и схемах.

## Consequences

- Пайплайн не воспроизводится одной командой: в LLM-шагах человек/агент в цикле.
- Метрика качества извлечения снимается вручную (счётчики ревизии), а не из логов.
- Пересмотр запланирован после гейта M2 — п. 2 пост-MVP бэклога в `PLAN.md`.
