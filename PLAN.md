# План — Miskatonic Atlas (рабочее название)

MVP зафиксирован 2026-07-13 по итогам grilling-сессии, закрыт 2026-07-14 (гейт M2 пройден). Фаза 3 «Подача» зафиксирована 2026-07-15 по итогам второй grilling-сессии; этап 6 «Полировка» добавлен 2026-07-18 по итогам третьей. Полная спека продукта: `docs/tech-spec.md`. Глоссарий: `CONTEXT.md`.

## Цель MVP

Вертикальный срез на одном рассказе — **«The Call of Cthulhu» (1928)**: полный путь «корпус → плейбуки извлечения → ревизия → провалидированный контент → статический сайт с картой», задеплоенный на Vercel. MVP отвечает на два вопроса:

1. **Как атлас выглядит и ощущается** — карта, страницы сущностей.
2. **Работает ли извлечение** — даёт ли пайплайн ≥70% полезных сущностей при ревизии ≤3–4 ч.

Публичного запуска в MVP нет.

## Ключевые отличия от спеки

| Спека | MVP | Причина |
|---|---|---|
| Anthropic SDK в `scripts/extract.ts` | Плейбуки в `prompts/`, исполняются в Claude Code | нет затрат на API до валидации идеи; см. ADR-0001 |
| Корпус — 6 рассказов | Только The Call of Cthulhu | срез, а не полное демо |
| Пайплайн (M1) раньше карты (M2) | Карта раньше пайплайна | цель «сначала увидеть»; координаты в любом случае ручные |
| События + таймлайн | Вне MVP; EventSchema заморожена в `schemas.ts` | отдельная сложность нормализации дат; CoC ими перегружен |
| Карта «Новая Англия + врезки» | Антикварный PD-скан карты мира | география CoC глобальна; скан бесплатен и легально чист |
| SEO/OG, аналитика, about + CTA | Вне MVP | инструменты запуска, а не чекпоинта |

## Зафиксированные решения

- **Язык сайта и контента — только английский.** `nameRu` в схемах остаётся optional, не заполняется.
- **Репозиторий** — этот каталог, git init, GitHub public (private создать не удалось; для MVP некритично — код MIT, цитаты PD), Vercel подключён к репо. Рабочее имя Miskatonic Atlas; настоящее имя и домен — решение к запуску.
- **Данные** — `content/*.json` + Zod; весь доступ к данным только через `src/shared/lib/content.ts` (мост к БД в продуктовой фазе).
- **Координаты карты** — в схеме сразу `mapId` (optional, default `"world"`), чтобы региональные карты пост-MVP не потребовали миграции контента.
- **Черновики пайплайна коммитятся** в `content/drafts/` — история качества извлечения пригодится как данные для метрики.
- **Версии стека** — актуальные стабильные на момент M0 (спека называла Next.js 15 — проверить, не хардкодить).

## Этапы MVP (закрыты 2026-07-14)

### M0 — Каркас (~1–2 вечера)

- git init, `.gitignore`, README: MIT для кода + секция о PD-статусе текстов с годами публикации
- Next.js (App Router, full SSG) + TypeScript strict + Tailwind
- `src/shared/schemas.ts`: все схемы из спеки, включая замороженную EventSchema; `CreatureSchema` через `.omit({role}).extend({classification})`; `map: {mapId?, x, y}`
- `src/shared/lib/content.ts`: чтение + Zod-валидация контента
- 2–3 локации из «Зова Ктулху» руками (Providence, R'lyeh, Sydney) с настоящими цитатами — валидны по `sources.min(1)`, лягут на карту в M1, пайплайн в M2 их обогатит
- Страницы `locations/[slug]` (generateStaticParams), простая главная-заглушка
- Деплой на Vercel

**DoD:** цикл «правка JSON → страница на проде» работает; валидация падает на битой ссылке.

### M1 — Карта (~3–4 вечера) — чекпоинт «как это выглядит?»

- Подбор подложки: 3–5 PD-сканов карты мира XIX века (Wikimedia Commons, David Rumsey, Библиотека Конгресса) → выбор одного → тонировка (сепия, затемнение) → 2048–4096 px
- **Координаты привязаны к картинке** — массовая расстановка только после выбора подложки
- Виджет WorldMap: react-leaflet, `L.CRS.Simple`, `ImageOverlay`
- Admin-кликер координат: dev-only роут, клик по карте → `{x, y}` для вставки в JSON
- Маркеры локаций, клик → панель-превью → переход на страницу локации
- Карта становится главной страницей

**DoD:** карта с маркерами ручных локаций на проде; Р'льех стоит по каноническим координатам из текста (S 47°9′, W 126°43′) по градусной сетке скана.

### M2 — Пайплайн (~1 неделя) — чекпоинт-гейт «извлечение работает?»

- `corpus/raw/`: текст CoC с Project Gutenberg, источник во фронтматтере; `scripts/normalize.ts` → `corpus/normalized/` (шапки Gutenberg убраны, пронумерованные абзацы)
- `prompts/extract.md` — плейбук извлечения: окна по абзацам с перекрытием; только локации/персонажи/существа (события — пост-MVP); правила: «extract only what is explicitly in the passage, no outside mythos knowledge», обязательная `evidenceQuote` + номер абзаца, имена as written + алиасы отдельным полем; выход — `content/drafts/`
- `scripts/verify-quotes.ts` — каждая `evidenceQuote` ищется точным поиском по normalized-тексту; несовпавшие помечаются
- `prompts/merge.md` — плейбук слияния: группировка по имени + алиасам, конфликтные поля → `NEEDS_REVIEW`
- **Ревизия** черновиков → `content/{locations,characters,creatures,stories}/`; счётчики с первого прогона: принято как есть / поправлено / мусор
- `scripts/validate.ts` на prebuild: Zod + целостность всех slug-ссылок
- Страница рассказа `stories/the-call-of-cthulhu` со сводкой и его сущностями; перекрёстные ссылки локации ↔ персонажи ↔ существа
- Координаты новых локаций через admin-кликер

**DoD:** сайт на проде целиком из извлечённого и отревизированного контента.

### Гейт после M2

- **≥70%** сущностей приняты как есть или с мелкой правкой **и** ревизия уложилась в **3–4 ч** → масштабировать корпус (бэклог)
- Иначе → чинить плейбуки, а не тащить объёмом; счётчики показывают, что именно ломается

**Результат (2026-07-14): пройден с запасом** — 95,6% принято (53 as-is + 12 edited из 68), ревизия ~1 ч. Путь на масштабирование корпуса открыт, но следующей выбрана Фаза 3 «Подача»: визуальный чекпоинт показал, что подача сырая, а извлечение навалило корректных, но проходных сущностей без иерархии.

## Фаза 3 — Подача (~2,5–3 недели с этапом 6)

Рамка: главный судья фазы — **автор-покупатель** будущего продукта «текст → библия мира». Атлас — витрина пайплайна: каждая фича отвечает на вопрос автора «что я получу для своей книги». Портфолио-эффект — попутный.

### Решения фазы

- **Подложка остаётся** — скан Colton 1852; генеративный арт — только во вклейках и декоре (ADR-0002). Иллюстрированная карта — это будущая региональная Новая Англия, не карта мира.
- **Значимость (Prominence)** — поле `prominence: "major" | "minor"` (default `major`) в схемах локаций, персонажей и существ; термин в глоссарии. Minor — unlisted: страницы собираются и открываются по перекрёстным ссылкам, но не попадают на карту, в Index и меню. Физически не удаляем ничего — «мир глубже карты».
- **Курация без UI** — решения по 65 сущностям списком в чате с рекомендациями, prominence проставляет скрипт (по образцу M2-ревизии). Ревью-UI — в бэклоге с якорем «строить перед следующим рассказом».
- **Арт-направление «тьма + бумага»** — тёмная оправа (хэдер, фон, виньетка вокруг карты), контент на пергаментных «листах», типографика издания XIX века.
- **Навигация — единый Index** — алфавитный указатель с секциями Locations / Characters / Creatures / Stories (только major). Хэдер: Map · Index.

### Этапы (по зависимостям)

1. **Курация** (~2–3 ч): prominence всем сущностям; `content.ts` — фильтры видимости (карта/Index показывают major, страницы собираются все).
2. **Стиль и типографика** (~2 вечера): шрифты эпохи, палитра «тьма + бумага», пергаментные панели, оформление цитат-источников; хэдер Map · Index, бейдж «MVP» убрать.
3. **Подача карты** (~3–4 вечера): главная — full-bleed карта на весь вьюпорт: хэдер — тонкая плавающая полоса поверх, футер на главной убран (PD-подпись остаётся мелкой строкой поверх карты); слева панель-легенда — название рассказа + список major-локаций, клик → пан/зум к пину + превью, на мобиле сворачивается; тонировка и затемнение краёв, кастомные пины и каллиграфические лейблы, декор, панель-превью в новом стиле. Внутренние страницы — с обычным хэдером/футером.
4. **Index** (~1 вечер): страница-указатель.
5. **Вклейки** (~2 вечера): 3–5 гравюр по образцу Plate I — Р'льех, болотный ритуал, «Алерт» против Ктулху, опционально барельеф Уилкокса; подпись с цитатой-источником (вклейка — UI-слой, в контент не попадает).
6. **Полировка** (~5–6 вечеров, зафиксирован 2026-07-18) — закрывает фазу:
   - **Маршруты плаваний** (флагман, ~1–1,5 вечера) — пунктирные треки в стиле морских карт XIX века: «Эмма» (Окленд → точка встречи) → «Алерт» (→ Р'льех) → дрейф Йохансена → «Виджилент» (→ Сидней), подписи судов вдоль линий. Маршрут — UI-слой по образцу вклеек (термин в глоссарии): определение в `src/widgets/world-map/`, точки редактор ставит по цитатам, цитаты-источники в подписи.
   - **Связность страница → карта** (~1 вечер): врезка-миникарта на страницах локаций — статичный фрагмент скана с пином по имеющимся `x,y`, без leaflet (врезка у minor-локаций допустима: это не вынос на общую карту, Значимость не нарушена); «View on the chart →» (`/?focus=slug`, только major).
   - **Превью-панель богаче** (~0,5 вечера): градусные координаты (пересчёт из пикселей по калибровке M1), миниатюра вклейки, «кто здесь встречается».
   - **Пины по типу локации** (~0,5 вечера): глифы город / остров / море.
   - **Страницы** (~1,5–2 вечера): буквица на первом абзаце + концовка ❦; Plate V — сгенерировать гравюру барельефа Уилкокса и подключить (каркас в коде готов); страница рассказа — галерея вклеек рассказа + маршрут; тематический фавикон + 404 на пергаменте.

**DoD этапов 1–5 (выполнен 2026-07-15):** на проде full-bleed карта с панелью-легендой и страницы в новом стиле; minor-сущности не видны на карте/Index/меню, но открываются по ссылкам; Index в хэдере; 3–5 вклеек.

**DoD этапа 6 (гейт закрытия фазы):** маршруты, врезки, деплинк, обогащённое превью, типовые пины, буквица/концовки, Plate V, страница рассказа, фавикон и 404 — всё на проде.

**Результат (2026-07-18): выполнен целиком** — маршруты (4 лега с цитатами, клик → панель-источник; концы по каноническим координатам через калибровку M1), врезки-миникарты с градусами, `/?focus=slug`, превью с градусами/миниатюрой/«кто здесь», глифы пинов, буквица (приподнятый инициал) + ❦, галерея вклеек и инсет маршрута на странице рассказа, фавикон, 404; Plate V (барельеф Уилкокса) сгенерирован редактором и подключён. Гейт закрывается пушем на прод.

### Вне фазы

Ревью-UI, новые рассказы, события/таймлайн, SEO/OG, аналитика, запуск, замена подложки, региональные карты.

## Бэклог после Фазы 3 (по порядку)

1. **Ревью-UI** — `/admin/review`, dev-only (как кликер координат), запись решений в JSON через route handler: режим ревизии черновиков (as-is/edited/junk + развилки needsReview) и режим курации (prominence). Строить перед следующим рассказом; это же — прототип продуктового «UI ревизии для автора» из спеки.
2. Решение о токенах: `scripts/extract.ts` через Anthropic SDK, те же промпты из `prompts/` (пересмотр ADR-0001)
3. Остальные 5 рассказов: Dagon (1919), The Nameless City (1921), The Festival (1925), The Colour Out of Space (1927), The Dunwich Horror (1929)
4. Региональная карта Новой Англии второй картой (`mapId` уже заложен) — вымышленная география, здесь уместна иллюстрированная/генерированная отрисовка
5. События + таймлайн на CoC (схема уже заморожена)
6. Декор карты: картуш — типографский заголовок в углу («Being a chart of the events of…»), без генеративного арта (ADR-0002)
7. SEO/OG (`next/og`, sitemap, JSON-LD), аналитика (Plausible/Umami), about + CTA + email-форма
8. Публичный запуск: r/Lovecraft, r/worldbuilding; настоящее имя и домен. Не раньше полного корпуса — единственный выстрел по комьюнити не тратим на один рассказ

## Явно НЕ делаем в MVP

Аккаунты, загрузка рукописей, БД, API/SDK, i18n и русская версия, события и таймлайн, OG/SEO-полировка, аналитика, граф связей (d3/sigma), изображения сущностей.

## Оценка

~2–2.5 недели при 10 ч/нед (полная спека оценивала M0–M4 в 4–5 недель).
